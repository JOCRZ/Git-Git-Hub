The first thing you should do when you install Git is to set your user name and email address.

git config user.name "John Doe"
git config user.email johndoe@example.com

If you use - -global, you only need to pass credentials once.

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com
There are two ways you can obtain a Git repository :
  • (Method 1) You can take a local directory that is currently not under version control, and turn it into a Git repository.

    Steps for Method 1:
    Go to Folder you want your local repository.

    cd C:/Users/user/my_project
    git init

    This creates a new subdirectory named .git that contains all of your necessary repository files. If you want to start version-controlling existing files (as opposed to an empty directory), you should probably begin tracking those files and do an initial commit.

     git add .
 git add LICENSE
 git commit -m 'Initial project version'
  • (Method 2) You can clone an existing Git repository from elsewhere.

    Steps for Method 2:

    git clone https://github.com//

    You can clone a repository with git clone and adding the git <url >at the end of command.

    If you want to clone the repository into a directory named something other than repo name, you can specify the new directory name as an additional argument :

    git clone https://github.com//   local_folder_name

Files on git has Two states Tracked & Un-tracked
StateDescription
TrackedFiles that are already part of the version control system and are being tracked for changes. These files have been previously committed, and Git is aware of their existence. Changes made to tracked files can be staged and committed.
UntrackedFiles that are not yet part of the version control system and are not being tracked for changes. These files are new or have been created but have not been added to the staging area or committed. Git is not aware of their existence, and they need to be explicitly added using git add before they can be staged and committed.

To create a new repository on the command line

echo "# newcase" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/JOCRZ/newcase.git
git push -u origin main

To push an existing repository from the command line

git remote add origin https://github.com/JOCRZ/newcase.git
git branch -M main
git push -u origin main
