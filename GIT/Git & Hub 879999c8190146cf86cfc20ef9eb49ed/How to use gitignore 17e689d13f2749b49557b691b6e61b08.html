How to use .gitignore

To ignore .csv files in your Git repository, follow these steps:

Step 1: Create a .gitignore file (if it doesn't exist):
If your repository doesn't already have a
.gitignore file, create one in the root directory of your Git repository. Use a plain text editor and save the file without any file extension.

Step 2: Add the .csv file pattern to .gitignore:
Open the
.gitignore file and add the following line to ignore all .csv files:

*.csv

This will instruct Git to ignore any file with the .csv extension.

Step 3: Save and commit changes:
Save the
.gitignore file after adding the pattern. Then, commit the changes to your repository using the following commands:

git add .gitignore
git commit -m "Added .gitignore to ignore .csv files"

Step 4: Verify that .csv files are ignored:
You can use
git status to check if the .csv files are now ignored. They should not appear in the list of modified files to be committed.

Any .csv file you add to the repository or modify after setting up the .gitignore will be ignored by Git, and you won't see them in the status or included in future commits. This keeps your repository clean and prevents unnecessary tracking of data files like .csv files, which are often generated and can change frequently.
