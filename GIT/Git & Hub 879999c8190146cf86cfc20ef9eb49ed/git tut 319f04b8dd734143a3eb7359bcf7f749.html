Git Scenario 1 cloning local folder on New git-hub repo ( Pushing exisisting repo to remote repo)

Step 1: git status # check all changes are commited
step 2: git push https://github.com/user/reponame.git master

Making Alias of the git remote url


git remote add origin https://github.com/user/reponame.git master

# so now you can use

git push origin master

Git Scenario 2 cloning New git-hub repo on Local folder ( Pulling exsisting remote repo to local repo)

# use cd and go to location you want your repo
git clone https://github.com/user/reponame.git 

# since we clone we don't have to add remote add to this origin is setted as default

#after chages and commit you can push to remote repo

git push origin master

To merge when you are working with a team

git pull origin main # before adding a new branch always pull from remote repo

git branch branch-name # you should create new branch instead of working on main branch

git checkout branch-name

# after editing files

git add .

git commit -m 'message'

git push origin branch-name


Step 2 : Process on Git-hub
On git-hub click on 'compare and  pull request' , this will ask permission
to other team members about merging the new branch to main after they have
checked or verified the new edit. ( done to limit conflicts ) The compare
and pull request will check the differences of new-branch and main branch.
